Architecture Note

The repo is intentionally structured like a small lending-risk stack instead of a single notebook.

Layers
01

Source Data

Application data, bureau history, previous applications, installments, POS, and revolving-credit tables feed one borrower-level master view.

02

Feature & Analysis

`features.py` builds the borrower feature base and `analysis.py` writes audit, target, drift, and proxy-review artifacts.

03

Scoring

`scorecard.py` builds the operating scorecard and `leaderboard.py` holds the challenger benchmark stack.

04

Presentation

`build_web_bundle.py` compiles the summary payload, borrower pages, plots, and lookup database used by the site.

Monitoring Loop
1Score borrowers with the explainable model.
2Monitor PSI, vintage, survival, calibration, and bad rate by decile.
3Inspect borrower traces for manual review or collections discussion.
4Re-bin, challenge, or refresh policy when thresholds are breached.
Fraud Extension

If extended for a blended credit/fraud team, add an upstream fraud decision layer for application velocity, identity consistency, bureau inquiry bursts, and device or session anomalies. That fraud layer should feed the same borrower trace surface.
