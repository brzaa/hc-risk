Overview

Framed as lending-risk work, not as a competition artifact.

The Home Credit data is used here as a public proxy lending book. The project is organized around origination policy, portfolio monitoring, explainability, and collections thinking rather than leaderboard-first model tuning.

Case Context

A multi-table consumer credit dataset is turned into a lending risk operating surface: what gets approved, what moves to manual review, what requires monitoring, and how one borrower is explained back to an analyst.

Decision Goal
OriginationRank, explain, and route applications
MonitoringTrack PSI, vintage seasoning, and calibration drift
CollectionsTranslate stressed traces into borrower review paths
Design Principle

The explainable scorecard is treated as the operating model surface. The ensemble remains a challenger and benchmark, not the default policy engine.

Vintage Monitor SQL

Warehouse-style query for cohort seasoning and cumulative 30+ DPD tracking across month-on-book. 

WITH booked_loans AS (
    SELECT
        loan_id,
        borrower_id,
        DATE_TRUNC('month', booked_at) AS booked_month,
        booked_at,
        approved_amount
    FROM risk.loan_originations
    WHERE booked_at >= CURRENT_DATE - INTERVAL '18 months'
),
first_severe_dpd AS (
    SELECT
        repayment.loan_id,
        MIN(repayment.mob) AS first_30dpd_mob
    FROM risk.repayment_events AS repayment
    WHERE repayment.days_past_due >= 30
    GROUP BY repayment.loan_id
),
loan_perf AS (
    SELECT
        loans.loan_id,
        loans.borrower_id,
        loans.booked_month,
        loans.approved_amount,
        COALESCE(events.first_30dpd_mob, 999) AS first_30dpd_mob,
        CASE WHEN events.first_30dpd_mob IS NULL THEN 0 ELSE 1 END AS severe_dpd_event
    FROM booked_loans AS loans
    LEFT JOIN first_severe_dpd AS events
        ON loans.loan_id = events.loan_id
),
mob_curve AS (
    SELECT
        perf.booked_month,
        mob.mob,
        COUNT(*) AS loans_in_cohort,
        AVG(
            CASE
                WHEN perf.severe_dpd_event = 1
                     AND perf.first_30dpd_mob <= mob.mob THEN 1.0
                ELSE 0.0
            END
        ) AS cumulative_30dpd_rate
    FROM loan_perf AS perf
    CROSS JOIN (
        SELECT 1 AS mob UNION ALL SELECT 2 UNION ALL SELECT 3 UNION ALL SELECT 4 UNION ALL
        SELECT 5 UNION ALL SELECT 6 UNION ALL SELECT 7 UNION ALL SELECT 8 UNION ALL
        SELECT 9 UNION ALL SELECT 10 UNION ALL SELECT 11 UNION ALL SELECT 12
    ) AS mob
    GROUP BY perf.booked_month, mob.mob
)
SELECT
    booked_month,
    mob,
    loans_in_cohort,
    cumulative_30dpd_rate
FROM mob_curve
ORDER BY booked_month DESC, mob;
Portfolio KRI SQL

Snapshot query for decision-band volumes, expected-loss proxy, booked exposure, delinquency, PSI, and calibration by decile. 

WITH current_scores AS (
    SELECT
        score_date,
        borrower_id,
        decision_band,
        score_decile,
        score_probability,
        expected_loss_proxy,
        approved_amount
    FROM risk.daily_score_snapshot
    WHERE score_date = CURRENT_DATE - INTERVAL '1 day'
),
portfolio_perf AS (
    SELECT
        perf.borrower_id,
        perf.booked_30dpd_flag,
        perf.booked_90dpd_flag,
        perf.current_dpd,
        perf.outstanding_principal
    FROM risk.portfolio_performance_daily AS perf
    WHERE perf.snapshot_date = CURRENT_DATE - INTERVAL '1 day'
),
psi_watch AS (
    SELECT
        monitor.feature_name,
        monitor.psi_value,
        ROW_NUMBER() OVER (ORDER BY monitor.psi_value DESC) AS psi_rank
    FROM risk.scorecard_feature_monitor AS monitor
    WHERE monitor.snapshot_date = CURRENT_DATE - INTERVAL '1 day'
),
calibration_view AS (
    SELECT
        decile,
        AVG(predicted_pd) AS avg_predicted_pd,
        AVG(observed_bad_flag) AS observed_bad_rate
    FROM risk.score_decile_monitor
    WHERE snapshot_date >= CURRENT_DATE - INTERVAL '30 days'
    GROUP BY decile
)
SELECT
    scores.score_date,
    scores.decision_band,
    COUNT(*) AS borrowers,
    AVG(scores.score_probability) AS avg_score_probability,
    SUM(scores.expected_loss_proxy) AS total_expected_loss_proxy,
    SUM(scores.approved_amount) AS booked_exposure,
    AVG(COALESCE(perf.booked_30dpd_flag, 0)) AS booked_30dpd_rate,
    AVG(COALESCE(perf.booked_90dpd_flag, 0)) AS booked_90dpd_rate,
    MAX(CASE WHEN psi.psi_rank = 1 THEN psi.psi_value END) AS max_feature_psi,
    MAX(CASE WHEN psi.psi_rank = 1 THEN psi.feature_name END) AS max_psi_feature
FROM current_scores AS scores
LEFT JOIN portfolio_perf AS perf
    ON scores.borrower_id = perf.borrower_id
LEFT JOIN psi_watch AS psi
    ON psi.psi_rank = 1
LEFT JOIN calibration_view AS cal
    ON cal.decile = scores.score_decile
GROUP BY scores.score_date, scores.decision_band
ORDER BY
    CASE scores.decision_band
        WHEN 'Approve' THEN 1
        WHEN 'Manual Review' THEN 2
        WHEN 'Decline' THEN 3
        ELSE 4
    END;
Architecture Note
Source DataApplication, bureau, previous application, installment, POS, and revolving-credit tables feed a borrower-level master table.
Feature Layer`features.py` and `analysis.py` produce monitoring-ready analysis artifacts reused by the web layer.
Scoring LayerThe WoE scorecard is the operating model; the ensemble remains the challenger benchmark.
Presentation`build_web_bundle.py` compiles summary JSON, plots, borrower pages, and the lookup database for the web UI.
Monitoring LoopScore borrowers, monitor PSI/vintage/survival/calibration, inspect traces, then re-bin or refresh policy when thresholds are breached.
Fraud LayerAdd upstream velocity, identity, bureau burst, and device/session checks while preserving one borrower trace surface.
